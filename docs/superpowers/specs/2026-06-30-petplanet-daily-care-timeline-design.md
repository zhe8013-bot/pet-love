# PetPlanet 日常照护时间轴设计

## 目标

在“生活记录”页增加一个完整的日常照护模块，把单次喂食、单次饮水、按需体重和生活照片收拢到同一块区域。保留原有月度消耗与花费能力，不要求每天记录体重，也不为照片强制建立每日相册。

## 页面结构

日常照护模块位于页面标题之后、本月消耗之前，使用当前奶油色与低饱和紫灰视觉语言。

1. 模块标题与日期切换：默认今天，可切换前后日期。
2. 四个快捷入口：记录喂食、记录饮水、记录体重、上传照片。
3. 当日时间轴：按发生时间倒序显示喂食和饮水记录。
4. 体重区：显示最新体重、较上次变化和现有趋势图；记录入口保留，体重不参与每日完成要求。
5. 照片区：汇总最近生活照片，照片不绑定某一天的记录卡。
6. 月度消耗区：继续保留在日常照护模块下方。

## 记录模型

### 喂食与饮水

每次行为独立记录：

- 类型：喂食或饮水。
- 时间：必填，精确到分钟。
- 数量：必填，喂食默认使用 g，饮水默认使用 ml。

时间轴卡片呈现图标、类型、数量和时间，并支持删除。数据按当前宠物隔离。

### 体重

继续使用现有体重数据和表单。独立的体重趋势模块移动到日常照护模块内部；体重可随时记录，不要求按日填写。

### 生活照片

上传表单包含：

- 一张或多张照片：必填。
- 拍摄日期时间：必填，可由用户自行选择。
- 一句话说明：选填。

生活照片复用现有回忆数据源，因此上传后同时出现在生活页照片区、首页照片统计和“回忆”页。系统为简化上传使用默认心情“安心”，不自动标记为闪光回忆。

## 数据与接口

新增 `CareEvent` 数据结构和仓库方法，覆盖本地仓库、HTTP 仓库与 SQLite API：

- `listCareEvents(petId, date)`
- `addCareEvent(event)`
- `removeCareEvent(id)`

删除宠物时同步删除其照护事件。照片继续使用现有 Memory API 和资源上传能力，避免维护两套照片存储。

## 交互与错误处理

- 新增成功后关闭弹窗并立即刷新对应区域。
- 表单缺少时间或数量时显示中文错误，不清空用户已填写内容。
- 数量必须大于 0。
- 加载失败时模块显示重试入口，不影响下方月度消耗区。
- 删除使用现有确认弹窗。
- 移动端四个入口为两列布局，弹窗可滚动，时间轴与照片区不产生横向溢出。

## 验收

- 可按当前宠物新增、查看和删除喂食/饮水记录。
- 两类记录包含时间与数量，且按时间倒序显示。
- 体重趋势与记录入口位于日常照护模块内部，原有体重数据和删除能力不丢失。
- 可上传照片并自选拍摄日期时间；照片同时出现在生活页和回忆页。
- 原有月度消耗、花费、筛选和删除功能保持可用。
- 390 × 844 视口无横向溢出，所有入口和弹窗可操作。
