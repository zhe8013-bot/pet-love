# PetPlanet 提醒管理归位设计

## 目标

首页保留提醒总览，但不再承担新增和管理；日常页集中处理待办与提醒，不新增主导航或独立页面。

## 页面职责

### 首页

- “今日关键提醒”最多展示按截止时间排序的 3 条提醒。
- 移除“添加待办”按钮和新增弹窗。
- 移除完成、稍后等管理操作，只保留提醒标题、描述和截止时间。
- 右上角提供“全部提醒”入口，跳转到 `/daily?section=reminders`。
- 继续显示全部提醒的完成统计，避免丢失总览价值。

### 日常页

- 在今日照护模块前增加“待办与提醒”区块。
- 展示当前宠物的系统提醒和自定义待办，保持原有截止时间排序。
- 提供新增待办、完成、稍后处理；自定义待办额外提供删除。
- 从首页入口进入时，提醒区块成为页面首要定位目标。
- 表单继续使用现有字段：待办事项、描述、截至时间。

## 数据边界

- 继续使用现有 `localStorage` 键：`petplanet:tasks:<petId>` 与 `petplanet:custom-todos:<petId>`。
- 当前宠物切换后，提醒列表同步切换。
- 复诊、驱虫、月度体重等系统提醒继续由宠物档案和健康记录派生，不复制数据。
- 不接后端，不增加提醒推送、循环规则或独立提醒页面。

## 验收标准

- 首页最多显示 3 条提醒且没有“添加待办”按钮。
- 首页“全部提醒”链接指向 `/daily?section=reminders`。
- 日常页可以新增、完成、稍后和删除自定义待办。
- 系统提醒不可误删。
- 当前宠物切换后，首页预览和日常提醒均同步更新。
- 既有测试、类型检查和生产构建继续通过。
