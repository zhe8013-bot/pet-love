# PetPlanet 当前示例提醒移除设计

## 目标

允许用户直接删除当前展示的自动示例提醒，同时保证宠物档案、病历、体重和其他业务数据不受影响。后续来源内容变化后生成的新提醒继续按正常流程出现。

## 行为

- 当前示例提醒和自定义待办都显示删除按钮。
- 删除自定义待办时，继续删除该条 `localStorage` 待办数据。
- 删除示例提醒时，只把当前这一条标记为“已隐藏”，不修改提醒来源。
- 首页预览和日常页列表同时过滤已隐藏的系统提醒。
- 隐藏记录按当前宠物隔离。

## 提醒身份

系统提醒使用来源内容生成隐藏标识：

- 档案提醒：提醒名称 + 提醒日期。
- 复诊提醒：病历 ID + 复诊日期。
- 月度体重：当前年月。

因此，档案提醒内容、复诊日期或月份发生变化时，会被视为一条新提醒并重新出现。

## 数据边界

- 新增 `petplanet:dismissed-reminders:<petId>` 本地存储键，只保存系统提醒隐藏标识。
- 不删除或修改宠物资料、病历、体重记录和 Repository 数据。
- 不接后端，不增加恢复中心或复杂提醒规则。

## 确认提示

删除系统提醒时提示：“只会从提醒列表中移除，不会删除宠物档案或病历。”

## 验收标准

- 原有系统提醒均可删除。
- 删除后首页和日常页不再显示该提醒。
- 刷新页面后保持隐藏。
- 切换宠物不会误隐藏其他宠物的提醒。
- 来源内容变化后，新提醒重新出现。
