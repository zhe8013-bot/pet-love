# PetPlanet 非 3D 产品完善设计

## 目标

暂停记忆星河、3D 与 2.5D 模块，把首页、宠物档案、健康档案和生活记录完善成视觉统一、逻辑完整、可持续使用的宠物照护产品。桌面与移动端均遵循 Stitch 的 Warm Lantern Garden 母版。

## 范围

### 全局框架

- 一级导航只显示：首页、宠物档案、健康档案、生活记录。
- 保留记忆模块代码和路由，当前不在导航、首页或快捷操作中出现。
- 全局顶栏提供当前宠物切换和快捷新增入口；快捷新增可打开新增病历、记录消耗和记录体重。
- 桌面使用固定侧栏，移动端使用四项底部导航。

### 首页

- 显示三只宠物的紧凑角色卡，可切换当前宠物并进入宠物档案。
- 今日待办由宠物提醒和待复诊记录组成；完成、稍后操作会即时更新并在本地持久化。
- 本月体重、消耗、最近健康和最近记录全部从 repository 数据计算，不使用与当前宠物无关的硬编码摘要。
- 快捷入口只指向当前可用的非 3D 功能。

### 宠物档案

- 新增 `/pets` 页面，展示当前宠物身份、基础资料、关键提醒、最近健康摘要、近六个月体重和本月照护统计。
- 支持多宠切换、编辑资料、上传头像、删除宠物，以及新增病历、记录体重和记录消耗。
- 删除宠物必须有明确二次确认；成功后自动选择剩余宠物。

### 健康档案

- 支持关键词搜索、状态筛选和时间范围筛选，并显示结果数。
- 病历记录展示诊断、症状、医院、费用、状态、复诊日期和照片。
- 支持详情弹层、删除确认、新增病历、加载状态、请求失败重试和无结果空状态。

### 生活记录

- 按月切换，展示总花费、主粮、饮水和护理次数。
- 显示近六个月体重趋势、消耗分类筛选、消耗列表和体重历史。
- 支持新增/删除消耗和体重；删除体重后当前体重按剩余最新记录重新计算。
- 空月份显示明确的新增入口。

## 数据与上传

- 所有页面只通过 `PetRepository` 访问数据。
- Repository 增加统一资源上传能力：本地模式转换为 data URL；API 模式调用 `/api/assets`。
- 新建宠物时先建立宠物记录，再上传头像并 patch；编辑宠物和新增病历先上传资源，再提交领域记录。
- 表单保存期间禁用提交按钮；失败时保留输入并显示中文错误。

## 状态与反馈

- 页面加载显示骨架或明确加载状态。
- 请求失败显示错误说明和重试按钮，不静默回退本地数据。
- 删除操作使用统一确认弹层。
- 新增、修改、删除成功显示轻量提示。
- 空数据、筛选无结果和无宠物三类状态分别处理。

## 验收

- 四个一级页面可从导航到达，所有可见按钮均有真实行为。
- 宠物切换后四页数据同步更新。
- 新增、编辑、删除、筛选、月份切换、错误重试和空状态均有自动化测试。
- 通过 1440×1024 与 390×844 浏览器流程和视觉 QA。
- `npm test`、`npm run build` 和 API 模式关键流程通过。
