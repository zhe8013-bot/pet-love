# PetPlanet 新版前端信息架构设计

## 目标

将现有 PetPlanet 前端从“首页、健康、生活、回忆、宠物”调整为“首页、日常、健康、回忆、档案”，让高频照护记录与低频资料管理分离。保留全部现有数据与记录能力，不引入后端、不新增复杂业务、不加载真实 3D。

## 已确认方案

采用增量语义迁移，不整页重写：

1. 先改导航、路由和兼容跳转。
2. 再把现有功能按页面职责重新排序。
3. 最后补齐首页总览、回忆概览和档案入口的轻量层级。

不采用仅改文案的方案，因为它会留下 `/life`、`/pets` 与新产品语言不一致的问题；也不采用重建 feature 目录和页面的方案，因为风险和改动面不符合“逐步调整、优先跑通”的要求。

## 路由与导航

主导航固定为：

1. 首页 `/`
2. 日常 `/daily`
3. 健康 `/health`
4. 回忆 `/memories`
5. 档案 `/profile`

兼容策略：

- `/life` 重定向到 `/daily`，保留原查询参数。
- `/pets` 重定向到 `/profile`，保留原查询参数。
- 数据系统、API 模式和 Roadmap 不进入用户导航。

## 全局层

### 当前宠物

顶部当前宠物切换继续由 `PetDataProvider` 管理。切换后首页、日常、健康、回忆和档案统一读取同一个 `currentPetId`，不增加第二套状态。

### 快速记录

保留一个全局“快速记录”按钮，菜单顺序为：

1. 记录喂食 → `/daily?new=feeding`
2. 记录饮水 → `/daily?new=water`
3. 记录体重 → `/daily?new=weight`
4. 记录消耗 → `/daily?new=consumption`
5. 新增病历 → `/health?new=1`
6. 添加照片 / 回忆 → `/memories?new=1`

这些入口只复用现有表单，不创建新的数据类型。

## 页面职责

### 首页

首页只负责判断“今天需要关注什么”和把用户送到正确页面，顺序为：

1. 欢迎语与当前宠物主卡。
2. 今日关键提醒：系统提醒、自定义待办、完成与稍后。
3. 快捷记录入口：喂食、饮水、体重、照片。
4. 今日照护进度：当天喂食次数、饮水次数、最近体重、照片数量。
5. 月度 Bento：体重、本月花费、健康记录、生活照片。
6. 回忆预览。

首页不展示完整历史列表，不承担消耗编辑、病历筛选或宠物资料编辑。

### 日常

现有生活页升级为日常页，继续复用当前实现，页面顺序为：

1. 今日照护与日期切换。
2. 喂食、饮水、体重、生活照片四个高频入口。
3. 当天喂食与饮水时间线。
4. 体重趋势。
5. 生活照片汇总。
6. 月度消耗与分类筛选。

查询参数 `new=feeding|water|weight|consumption|photo` 对应打开现有表单。

### 健康

健康页保持现有职责和结构：健康概览、病历筛选、病历时间线、病历详情、照片、复诊、用药与删除。只调整导航位置，不扩展医疗业务。

### 回忆

回忆页以 2D 成长相册为唯一真实展示模式，顺序为：

1. 成长概览：照片数、陪伴天数、闪光回忆数。
2. 闪光回忆预览。
3. 季节时间线提示。
4. 2D 相册网格。
5. 添加回忆。
6. 未来 3D 记忆星河入口卡。

当前不加载 `MemoryUniverse`，不渲染 WebGL；现有 3D 代码保留但不进入用户主流程。

### 档案

现有宠物页升级为档案页，顺序为：

1. 当前宠物完整档案。
2. 身体信息与关键提醒。
3. 编辑资料和上传头像。
4. 添加宠物、删除宠物。
5. 新增病历、记录体重、记录消耗快捷入口。
6. 最近健康、体重和本月照护摘要。

添加宠物由档案页内的 `PetForm` 承担；侧栏入口改为指向 `/profile?addPet=1`。

## 数据与兼容性

- 继续使用 `PetRepository` 接口以及 localStorage/mock 数据。
- 不修改已有存储 key，不迁移或清空用户数据。
- 不新增服务端表和 API。
- 所有新总览数据由现有 `careEvents`、`weights`、`consumptions`、`medicalRecords`、`memories` 聚合得出。
- 测试夹具不再依赖真实系统月份，避免跨月后测试失效。

## 视觉规则

- 沿用奶油白、柔雾橘、灰紫、深墨紫。
- 沿用现有圆角、阴影、Bento 和 2.5D 浮动卡片，不建立第二套设计系统。
- 图标继续使用 Phosphor。
- 首页强调总览，日常强调操作，健康强调时间线，回忆强调照片，档案强调身份资料。
- 移动端保持五栏底部导航，顺序与桌面一致。

## 错误与空状态

- 旧路由必须可访问并自动转向新路由。
- 页面加载失败继续使用现有重新加载提示。
- 没有当天照护、没有照片、没有病历和没有消耗时继续显示对应空状态。
- 快速记录菜单打开表单失败时由现有表单错误状态负责反馈。

## 验收标准

- 桌面与移动端导航均为：首页、日常、健康、回忆、档案。
- `/daily` 和 `/profile` 可直接访问，旧 `/life`、`/pets` 可兼容跳转。
- 全局快速记录六个入口都能打开正确表单。
- 首页不再承担完整记录管理，只展示提醒、进度、快捷入口、月度概览和回忆预览。
- 日常页保留喂食、饮水、体重、照片、消耗的全部现有能力。
- 回忆页默认只有 2D 相册和未来 3D 入口，不加载真实 3D。
- 档案页可以添加、编辑和删除宠物。
- 切换宠物后所有页面内容同步。
- localStorage 中已有数据不丢失。
- 全量测试、生产构建和 390×844 可视检查通过。
